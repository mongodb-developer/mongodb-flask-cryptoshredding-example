Welcome!
This application demonstrates how you can use MongoDB's Client Side Field Level Encryption to effectively "forget" customer data - even if the data gets restored by a backup after being deleted.
When you sign up a new user, a new data encryption key will be generated and linked to the user. When you log in with this user, you can start adding data into the database. We'll use the generated encryption key to encrypt the data you save. It will be stored in a collection called data, structured as such: 
            
{
  "username": <your username (encrypted)>
  "name": <the name you enter>,
  "value": <the value you enter (encrypted)>
}
Visit the admin page, where you can trigger a bunch of different actions to test - for example deleting the data encryption key for your user. Deleting the key means the data you added can no longer be decrypted. This means the data can no longer be linked to your user, nor can its value be read.
To get started, sign up with a user.
